# 운동에 대한 강박을 없애는 개인화 AI Agent 헬스 캐어 서비스 'OMT' 기획서

# 페인 포인트: 꾸준하게 동기부여가 필요한 사용자를 위한 설계 부재 // 솔루션: 동기부여/꾸준함을 유발할 수 있는 기능

- 타깃 정의
  - 다이어트와 체형 관리에 관심은 있지만, **직장/학교 생활로 인해 작심삼일을 반복하는 20–30대 여성**
  - 헬스장/홈트/다이어트 앱을 한 번 이상 시도해봤지만, **1주~1달 이상 유지하지 못한 경험이 있는 사용자**
- 핵심 페인 포인트
  - “운동의 필요성”과 “의지”는 반복적으로 생기지만,
    - 현실적인 일정·피로·귀찮음 때문에 **계획 → 실행 → 유지**로 이어지지 못함
    - 기존 앱은 숫자/그래프/루틴/리워드 중심이라 **심리적 허들을 낮춰주지 못함**
    - 실패 시 다시 시작할 수 있도록 **전략을 재설계해주는 구조가 부재**
- 솔루션 방향
  - “완벽한 운동/식단 관리”가 아니라, **“계속 포기하더라도 다시 시작하게 만드는 대화형 AI 코치 + 가벼운 미션 구조”**
  - 운동(미션)을 서비스의 **메인 코어 루프**로 두고, 식단은 “행동 전략을 조정하는 보조 인사이트” 역할로 한정

## 앱스토어(건강 및 헬스케어 무료 앱 상위 25개 분석) 요약

- 데이터/기록/리포트, 외적 리워드, 전문 코칭, 미용/진료 등 **기능 단위로는 이미 시장이 포화**
- “습관/동기부여” 카테고리도 존재하지만:
  1. 플랜핏/짐워크: **명확한 목표와 의지가 있는 사용자**를 대상으로, 강도/효율 최적화에 집중
  2. 찰리/인아웃: **걷기/식단 기록** 중심으로, “심리적 허들·실패 패턴·생활 리듬”에 대한 다룸은 상대적으로 약함
- 결론:
  - **“계획–실행–실패–재시도” 루프를 다루는 서비스는 부족**하며, 여기에서 **대화형 AI 코치 + 최소 미션 구조**가 차별화 포인트가 될 수 있음

## PMF 가설: 초심자가 꾸준하게 운동/다이어트를 할 수 있도록 도와주면 운동/식단 관리에 대한 허들을 낮출 수 있을 것이다.

- 타깃 PMF 정의(리파인)
  - “다이어트·체형 관리에 관심은 있지만, 직장/학교 생활 때문에 작심삼일을 반복하는 20–30대 여성이 **대화형 AI 코치와 함께 ‘현실적인 운동 미션’을 수행할 수 있다면**, 기존 운동/다이어트 앱 대비 **N주차 유지율과 주당 활동일 수가 유의미하게 높아질 것이다.**”
- 서비스 전략
  - 운동(미션)을 **코어 루프**로 정의하고, 식단은 “행동 전략을 조정해주는 인사이트” 수준으로 **보조 기능**에 한정
  - AI는 단순 “정보 제공/기록 요약”이 아니라, **“실패 패턴을 해석하고, 다음 시도를 설계해 주는 코치”** 역할을 담당

### PMF 검증 질문

- “운동(미션)만 잘 설계해줘도 꾸준함이 생기는가?”
- “운동(미션)+식단을 묶으니 오히려 이탈이 늘어나는가?”
- 정량 관점
  - D1/D7/D14/4주차 잔존
  - 유저당 주간 미션 수행일 수
  - 식단 입력 유도 여부에 따른 잔존 및 DAU 변화
- 정성 관점
  - 2~4주 사용자를 대상으로:
    - “이 앱을 삭제하지 않은(또는 다시 돌아온) 이유는?”
    - “특히 귀찮은 날에 실제로 도움이 되었다고 느낀 순간은?”
    - “AI(대화형 코치)가 없었으면 어땠을 것 같은지?”

## MVP 스코프 정리 (필수 vs 후순위)

### v0 MVP – 필수 범위

1. **대화형 AI 코치 기반 데일리 미션 추천**
   - 아침/점심 1회 간단 체크인(질문 1~2개 + 버튼/퀵 리플라이)
   - 오늘 컨디션/일정에 맞는 **최소 운동 미션 2~3개** 제안
2. **저녁 회고 + 실패 패턴 기반 피드백**
   - 오늘 미션: [성공] / [실패]
   - 실패 이유 선택(피곤/야근/약속/잊어버림 등)
   - AI 코치가 패턴을 요약하고, **다음 시도 전략 수정 제안**
3. **캐릭터의 단순 상태 변화**
   - 캐릭터 상태: 지침 / 보통 / 활기참 / 성장 단계 (3~4단계)
   - 미션 성공/실패에 따라 상태 변화
4. **홈 화면 원터치 미션 시작**
   - 푸시/채팅에 답하지 않아도, 홈에서 “오늘 최소 미션 시작하기” 버튼으로 바로 실행 가능
5. **텍스트 기반 식단/상태 라벨링 (옵션)**
   - “오늘 식단 어땠어요?” 수준의 선택형/간단 입력
   - 내부적으로 패턴 분석/난이도 조정에만 활용

### v1 이후 – 후순위/로드맵

- 소규모 스쿼드(2~4인 그룹 + AI 그룹 요약)/커뮤니티
- 주간/월간 고급 리포트(자세한 패턴 그래프 등)
- 식단 세부 인사이트 강화(카테고리 확장, 더 정교한 피드백)
- 간헐적 강화/랜덤 미션, 테마 주간 등 리텐션 강화 기능
- A/B 실험 풀셋 자동화, 세그먼트별 실험(추후 트래픽 확보 후)

## 메인 기능: 유저 동인(꾸준함/동기부여 유발 기능)

### 1) 대화형 AI 코치 기반 데일리 미션 추천

- 매일 1회(아침/점심) 대화형 체크인
  - 오늘 컨디션: [괜찮음] [보통] [완전 피곤]
  - 오늘 일정: [야근] [약속 있음] [여유 있음]
- 입력값을 기반으로, **상황에 맞는 최소 미션** 2~3개 제안
  - 예:
    - A: 자기 전 스트레칭 5분
    - B: 퇴근길 5분 빠르게 걷기
    - C: 오늘은 운동 대신 야식만 줄이기
- 기초대사량/활동량 등의 수치 계산은 **내부 난이도 조정 로직**으로만 사용
  - UI에는 “오늘은 이 정도면 충분해요” 같은 질적 메시지로 표현

### 2) 저녁 회고 & 실패 패턴 기반 피드백

- 저녁/취침 전 간단 회고:
  - 오늘 미션: [성공] [실패]
  - 실패 시 이유:
    - [너무 피곤해서 잠] [약속이 늦게 끝남] [귀찮아서 미룸] [그냥 잊어버림]
- AI 코치가 패턴 요약 + 전략 수정 제안
  - 예:
    - “야근 있는 수요일은 실패율이 높네요. 수요일은 휴식일로 두고, 목요일에 5분 미션으로 옮겨볼까요?”
    - “자꾸 잊는다면, 퇴근 시간에 맞춰 알림을 조금 앞당길게요.”

### 3) 캐릭터를 통한 시각적 피드백

- 캐릭터는 사용자의 **습관/컨디션 상태를 시각화하는 미러**
  - 상태: 지침 → 보통 → 활기참 → 성장 단계
- 미션 성공 시:
  - 캐릭터 기분/상태가 한 단계 좋아지고, 짧은 칭찬 문구 제공
- 실패가 반복될 경우:
  - 캐릭터가 살짝 지친 모습 + 부드러운 피드백
  - 예: “우리 둘 다 요즘 많이 바쁘네요. 이번 주는 더 짧게 가볼까요?”

### 4) 대화형 UX 설계 원칙

- 한 번의 인터랙션(푸시 또는 앱 방문)당 **질문 최대 1~2개**
  - 추가 정보는 버튼/퀵 리플라이로 처리
- 온보딩은 “필수 질문 / 나중에 물을 질문”으로 분리
  - 필수:
    - 대략적인 운동 경험(거의 안 함/가끔/주 3회 이상)
    - 선호 형태(걷기/집 운동/헬스장)
    - 최소 가능 시간(5/10/20분)
  - 이후:
    - 체중/체형, 식습관, 수면 등 민감/상세 정보
- 푸시/알림에 반응하지 않아도 서비스가 깨지지 않게 설계
  - 홈 화면의 **“오늘 최소 미션 시작” 버튼**을 통한 진입 허용

## 추가 기능 가설

### 가설 1) 커뮤니티(식단/루틴 공유) 기능이 서비스 사용 동인이 될 것인가

- 가설 정의
  - “소규모 그룹/친구와의 약한 연결 + AI 요약이 혼자 사용할 때보다 **꾸준함과 정서적 지지감**을 높일 것이다.”
- MVP 단계에서의 위치
  - v0에서는 **필수 아님(후순위)**
  - 로드맵 상 컨셉: 2~4명 단위 스쿼드, 각자의 미션 성공 여부 공유, AI의 주간 요약
- 1차 PMF 검증 이후, 필요성/리소스를 고려해 v1+에서 구현 검토

### 가설 2) 텍스트 기반 식단 기록이 유저에게 필요할 것인가

- 방향
  - 이미지 기반 정밀 영양소 인식은 v0에서 제외
  - **텍스트·선택형 라벨링**으로 최소 정보만 수집
    - 예: “오늘 식단은 어땠어요?”
      - [과식] [탄수 위주] [단백질 위주] [야식] [+ 직접 입력]
- 역할
  - 식단 기능은 “정밀 관리 도구”가 아니라 **“운동 미션 난이도/전략을 조정하는 참고 인사이트”**
- 검증 포인트 (1차 실험 대상)
  - 운동만 제공 vs 운동+간단 식단 입력 유도
  - 식단 입력 유도가 **오히려 피로와 이탈을 늘리는지** 여부

### 가설 3) AI 코치 기능이 유저에게 필요할 것인가

- AI 코치 역할
  1. 현실적인 플래너: 일정/컨디션 기반 최소 미션 제안
  2. 실패 패턴 분석가: 요일·상황별 실패 패턴을 발견하고 전략 수정
  3. 요약형 Q&A 가이드: 초심자 질문에 대해 1분 내 읽을 가이드 제공, 노트화
- 2차 실험 대상 (트래픽 확보 이후)
  - “고정 미션 앱 vs 대화형 AI 코치 앱” 비교

### 가설 4) 캐릭터 키우기 기능이 꾸준함에 기여할 것인가

- 역할
  - 캐릭터 = “나의 습관/컨디션 상태를 보여주는 시각화 + 가벼운 동기 부여”
- MVP 범위
  - 성장 트리/아이템/스킨 대신, 상태 3~4단계 중심
- 1차 실험 대상
  - A: 캐릭터 없는 텍스트 피드백
  - B: 캐릭터 상태 변화 포함
  - 미션 완료 후 보람/재미·루프 반복률 비교

## 코어 루프 및 동기부여 설계

1. **즉각 보상 (Day 1부터 체감)**
   - 미션 완료 시:
     - 캐릭터 상태 변화
     - “오늘 4주 목표 중 1일을 채웠어요.” 같은 진행도 카피
     - 짧고 긍정적인 피드백
2. **축적 보상 (주/월 단위)**
   - 주간/월간 요약:
     - 미션 수행일 수
     - 가장 힘들었던 요일/상황
     - 지난 기간 대비 개선된 점
   - “습관이 쌓이는 느낌”을 캐릭터 변화 + 텍스트로 전달
3. **간헐적 강화/랜덤 요소 (로드맵)**
   - v0에서는 코어 루프(데일리 미션/회고) 검증에 집중
   - 이후 리텐션 데이터에 따라:
     - 랜덤 미션
     - 테마 주간(예: 수면/스트레스 집중) 등 적용 검토

## UX/카피 톤 가이드 (몸매/다이어트 압박 관련)

- 지양할 표현
  - 체중/사이즈를 직접 평가/비교하는 표현:
    - “지금은 비만입니다”, “몇 kg 빼야 합니다”
  - 외모 비교:
    - “누구처럼 날씬해질 수 있어요”
  - 죄책감 유도:
    - “오늘도 실패했네요”, “의지가 부족해요”
- 지향할 표현
  - 건강/컨디션/에너지 중심:
    - “오늘은 몸이 좀 무거웠죠. 그럴 땐 5분만 움직여도 충분해요.”
  - “완벽함”이 아닌 “한 걸음”:
    - “완벽한 하루보다, 어제보다 한 걸음만 더 나아가면 충분해요.”
  - 자기 수용 + 현실적인 제안:
    - “요즘 일정이 많이 바쁘네요. 이번 주는 더 짧은 미션으로 가볼게요.”

## AI 기능 및 정책 (요약)

- 텍스트 라벨링 중심
  - 식단/상태: 텍스트·선택형 라벨을 통해 수집
  - 카테고리(과식/야식/탄수 위주/단백질 위주 등)로 정규화해 패턴 분석
- 숫자/의료 정보 노출 최소화
  - BMR/TDEE 등은 내부 로직에만 사용
  - “오늘은 이 정도면 충분해요” 같은 질적 표현으로만 안내
- 법적/윤리적 책임
  - 온보딩·설정·주요 피드백 화면에 공통 문구:

# **LangGraph 및 Graph DB 정리**

1. **LangGraph**
   - 이번 서비스 구조(대화형 + 멀티 역할 AI + 상태 기반 플로우)에는 꽤 잘 맞는다.
   - 특히:
     - 온보딩/데일리/회고/요약 플로우 분리
     - 미션 추천/패턴 분석/FAQ 에이전트 분리
   - 추천: v0부터 써도 됨. 다만 노드 수를 최소한으로 유지하는 게 중요.
2. **그래프 DB**
   - “언젠가 쓰면 좋을 것 같은 기술”에 가깝다.
   - MVP 단계에서는:
     - RDB로 충분히 패턴 분석/통계를 낼 수 있고,
     - LangGraph가 이미 “플로우 그래프” 문제를 해결해 준다.
   - 추천:
     - v0: PostgreSQL 등 RDB + 그래프 친화적 스키마
     - v1 이후: 패턴/추천이 복잡해질 때 동시 도입 고려
3. **실질적인 조언**
   - 지금은 **“LangGraph로 대화 플로우와 에이전트 역할을 먼저 안정화”**
   - 데이터 레이어는 **RDB로 시작하되, 노드/엣지 관점을 머릿속에 두고 설계**
   - 그래프 DB는:
     - 유저가 어느 정도 쌓이고
     - “패턴 기반 추천/세그먼트 실험”을 진지하게 할 시점에 도입하는 게 비용 대비 효과가 좋다.

**해결 방안:**

**질문**: 온보딩에서 얼마나 많은 정보를 받을 건가요? (너무 많으면 이탈 위험)

### 2. **AI 응답 품질 관리**

```
문제: LLM이 가끔 이상한 답변
→ "오늘 파스타 10인분 먹었어"
→ AI: "좋았겠다! 내일도 화이팅!"

또는 민감한 상황 처리
→ "요즘 우울해서 먹기만 해"
→ AI: ???

```

**해결 방안:**

- Guardrails 필요 (이상 입력 감지)
- 민감 키워드 감지 시 전문가 상담 안내
- 응답 템플릿 + LLM 조합 (완전 자유 생성은 위험)

**질문:** AI가 100% 자유롭게 생성? 아니면 템플릿 기반 + 약간의 변형?

### 3. **미션 선택의 역설**

```
A안, B안, C안, D안 제시
→ 유저: "음.. 뭐하지?" (선택 피로)
→ 또는: "다 귀찮은데.." (회피)

```

**개선 아이디어:**

```
기본: AI가 1개 추천

유저: "너무 힘들어"
→ 더 쉬운 옵션 3개 제시

유저: "너무 쉬운데?"
→ 더 어려운 옵션 3개 제시

```

이렇게 **대화형으로 난이도 조절**하는 게 더 자연스러울 수도?

**질문:** 처음부터 4개 제시 vs 1개 추천 후 필요시 추가 옵션?

### 4. **캐릭터 시스템 디테일**

```
❌ 캐릭터 건강도 하락 (죄책감 유발)
✅ 캐릭터 "기다림" 상태 (돌아오길 기다리는 느낌)

❌ 연속 기록 끊기면 처음부터
✅ 총 누적 기록으로 캐릭터 성장
```

# AI 기능 및 기술·리스크 고려사항

- 텍스트 라벨링 중심 설계
  - 식단/상태 입력은 **텍스트 + 선택형 라벨** 중심
  - 카테고리(과식/야식/탄수 위주/단백질 위주 등)를 바로 업데이트하여 패턴 분석 및 미션 난이도 조절에 활용
- 숫자/의료 정보 노출 최소화
  - BMR/TDEE 등 수치는 내부 로직에만 사용
  - UI에는 “오늘은 이 정도면 충분해요” 수준의 **질적 메시지**로 표현
  - 프롬프트/정책 레벨에서:
    - 병명/진단/치료/의료적 위험도 직접 언급 금지
- 법적/윤리적 책임 회피 문구
  - 온보딩 및 설정, 주요 피드백 화면 등에 명시:
    - “본 서비스는 의료 조언이 아니며, 건강 상태에 따라 전문의와 상담하는 것을 권장합니다.”
  - 건강/다이어트 관련 조언은 **일반적인 생활 가이드 수준**으로 제한
